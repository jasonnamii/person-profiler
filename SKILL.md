# Person Profiler — 인물 프로파일링 엔진

## §1 PURPOSE

웹 공개정보 기반으로 인물을 다각도 분석하여, **"이 사람이 누구인지 30초 안에 설명할 수 있는"** 프로필을 생성한다.

핵심 질문: **어떤 사람이고 → 무엇을 하는 사람이고 → 요즘 어떤 상황이고 → 어떻게 접근해야 하는가**

## §2 PIPELINE

```
P1 프레임   →  P2 리서치   →  P3 프로파일링  →  P4 엘베피치  →  §3.5 리포트 변환
(목적·축설정)   (축별 웹조사)   (DNA·강약점도출)   (수렴→30초피치)   (3막 서사 변환)
```

| Phase | 핵심 동작 | 스킬 호출 | 산출물 | 실패 조건 |
|-------|----------|----------|--------|----------|
| P1 | 대상+목적 확인 → 7축 중 적용축 선정 | — | 분석 프레임 | 대상·목적 미확인 |
| P2 | **Skill("research-frame")** → 축별 독립 웹 리서치 | **research-frame** 필수 | 축별 발견 정리 | 스킬 미호출 / 소스 1개 이하/축 |
| P3 | 패턴 추출 → DNA·강약점·의사결정 도출 | — | §3 중간 산출물 | 증거 없는 DNA |
| P4 | Layer 1→2→3 수렴 → **Skill("trigger-dictionary")** 엘베피치 | **trigger-dictionary** (엘베피치) | 엘베피치 | 피치 미생성 |
| §3.5 | §3 중간 산출물 → 3막 서사 템플릿으로 변환 | — | **최종 리포트** | §3 직접 출력 / 서사 톤 미준수 |

### P1 프레임 설정

**GATE**: 대상 인물명 + 분석 목적 확인 → 미제공시 질문

**축 선정 규칙:**

| 축 | 핵심 질문 | 기본 | 조건 |
|----|----------|------|------|
| ① 경력 DNA | 뭘 해온 사람인가 | ✓ | 항상 |
| ② 의사결정 패턴 | 어떻게 결정하는가 | ✓ | 항상 |
| ③ 비전·방향성 | 어디로 가려는가 | ✓ | 항상 |
| ④ 평판·호불호 | 남들은 어떻게 보는가 | ✓ | 항상 |
| ⑤ 개성·특이성 | 이 사람만의 색깔은 | ✓ | 항상 |
| ⑥ 관계망·영향력 | 누구와 엮여 있는가 | — | 비즈니스·정치 목적 |
| ⑦ 현재 컨텍스트 | 요즘 뭐 하는가 | ✓ | 항상 |

- 기본 6축(①~⑤⑦) 자동 적용. ⑥은 목적이 비즈니스 미팅·파트너십·정치 접근일 때 추가
- 형이 축을 명시 지정하면 그대로 따름

### P2 웹 리서치

**⚠ MANDATORY: P2 진입 시 반드시 `Skill("research-frame")` 호출하여 리서치 프레임워크를 로드한다.**

리서치 프레임 로드 후 실행 순서:
1. research-frame §1 WHY: 인물 분석 목적 → 가설 설정 → 7축을 조사 프레임으로 전달
2. research-frame §2 HOW: 축별 독립 웹 리서치 실행 (아래 가이드 적용)
3. research-frame §3 SCOPE: 축별 소스 충분성 판정 → 종료 조건 확인

**축별 조사 규칙:**
- 축별 독립 조사 (축 간 오염 방지)
- 소스 우선순위: 공식(IR·정부·학술) → 전문(업계지·인터뷰) → 일반(기사·SNS)
- 축당 최소 3개 독립 소스. 2개 이하 → 해당 축 확신도 자동 하향
- **이미지 수집**: 웹서치로 인물 대표 이미지 URL 1~2개 확보 (공식 프로필·언론 사진 우선)

**축별 조사 가이드:**

| 축 | 검색 키워드 패턴 | 핵심 수집 대상 |
|----|----------------|---------------|
| ① 경력 | "[인물] 경력 이력 career" | 직함 변천, 재직기간, 이직 패턴 |
| ② 의사결정 | "[인물] 결정 선택 전략 decision" | 핵심 의사결정 사례 3+ |
| ③ 비전 | "[인물] 비전 목표 방향 vision plan" | 공식 발언·인터뷰·기고 |
| ④ 평판 | "[인물] 평가 논란 reputation 호불호" | 긍정·부정 평가, 논란 이력 |
| ⑤ 개성 | "[인물] 스타일 특징 성격 personality" | 말투·행동 패턴·취미·일화 |
| ⑥ 관계망 | "[인물] 네트워크 협업 partner" | 핵심 인맥, 소속 조직 |
| ⑦ 현재 | "[인물] 최근 2024 2025 현재" | 최신 뉴스·동향·프로젝트 |

### P3 프로파일링

**DNA 도출 규칙:**
- 전 축 조사 결과에서 **반복 패턴** 추출 → DNA 5~7대 명명
- 각 DNA: 명칭(한 단어~구) + 증거(구체적 사례 1+) + 강도(⭐1~5)
- 증거 없는 DNA 금지. 추론만으로 도출 시 "추정" 태그 필수

**강점·약점 판정:**
- 전 축 분석 결과를 수렴하여 강점 3~5개, 약점 3~5개 도출
- 각 항목: 1줄 판정 + 근거 축 번호 명시
- 강점과 약점이 동전의 양면인 경우 명시 (예: "추진력 ↔ 독단")

**의사결정 패턴:**
- 핵심 의사결정 사례 3개 이상에서 공통 루프 추출
- 표현: 텍스트 플로우 or 테이블 (Mermaid는 형 요청시만)

### P4 엘베피치 도출

**3-Layer 수렴:**

```
Layer 1 (Who)  : 약력·경력·성과 → "이 사람은 ___하는 사람이다"
Layer 2 (Why)  : DNA·패턴·강약점 → "특히 ___한 특징이 있다"
Layer 3 (So What): 현재 상황·접근법 → "지금 ___이고, ___하면 된다"
```

**엘베피치 포맷:**
- 30초(~100자 내외) 구두 설명 가능한 길이
- 구조: [누구] + [뭘 하는 사람] + [핵심 특징] + [지금 상황] + [접근법]
- 단정적 톤. 헤지 금지

## §3 분석 중간 산출물

P1~P4 파이프라인의 **분석 결과물**. 이 단계는 데이터 구조이며 최종 리포트가 아니다.

```
§0 이미지 앵커
   - 인물 대표 이미지 (웹서치 수집 URL, ![alt](url) 형식)
   - 이미지 미확보시 생략 (빈 칸 금지)

§1 엘베피치 (30초 버전)
   - Layer 1→2→3 수렴 결과
   - 전체 리포트의 스파인

§2 기본 프로필
   - 이름 | 직함·소속 | 학력 | 핵심 경력(5줄 이내)

§3 DNA 프로파일링
   - 5~7대 DNA, 각각: 명칭 | 증거 | 강도(⭐)

§4 의사결정 패턴
   - 핵심 사례 3+ → 공통 루프 도출
   - 리스크 수용도: 높음/중간/낮음 (영역별)

§5 강점·약점 판정
   - 강점 3~5개 | 약점 3~5개
   - 동전의 양면 관계 표기

§6 현재 컨텍스트 & 방향성
   - 최근 동향 (시간순 3~5개) + 향후 예상

§7 설득 포인트 / 접근 전략
   - High Fit 3개 | Low Fit 3개 | 추천 접근법

§8 리스크 & 맹점
   - 분석 한계 (소스 편향·정보 공백)
   - 인물 자체 리스크
```

이 중간 산출물은 **내부 분석용**이다. 형에게 전달하는 최종 리포트는 §3.5에서 변환한다.

## §3.5 리포트 변환

**MANDATORY**: P4(엘베피치) 완료 후, 위 §3 분석 결과를 아래 3막 템플릿에 맞춰 재구성하여 최종 리포트를 생성한다. §3의 중간 산출물을 그대로 출력하는 것은 금지.

### 톤 & 문체 규칙

| 규칙 | 이유 |
|------|------|
| **산문 우선** — 테이블은 프로필 카드·동향 타임라인에만 허용. 나머지는 문장 | 테이블 과잉 = 코드 집합 |
| **콜아웃 절제** — `[!tip]`(프롤로그) + `[!warning]`(에필로그) 2개만 | 콜아웃 남발 = 시각 노이즈 |
| **헤더 3단계** — `#`(제목) `##`(막) `###`(장면)까지. `####` 이하 금지 | 헤더 과잉 = 문서 파편화 |
| **증거는 각주식** — "(매체명, 날짜)" 형식으로 서술에 녹인다 | 별도 소스 섹션 불필요 |
| **단정적 톤** — 확신도가 낮으면 낮다고 명시하되 문장은 단정 | 프로파일링은 판단 |
| **불릿 금지** — 나열은 산문으로. "세 가지가 눈에 띈다: A, B, C." | 불릿 = 읽히는 글의 적 |

### 3막 템플릿

```
# [인물명] 프로파일

> [!tip] 프롤로그 — 30초 엘베피치
> (§1 엘베피치 전문. 리포트 전체의 스파인.)

---

## 1막: 누구인가

프로필 카드 (테이블 1개: §2 기본 프로필)

이어서 §2 경력을 서사로 풀어쓴다. 연표 나열이 아니라
"어디서 시작해서 → 어떤 전환점을 거쳐 → 지금 어디에 서 있는가"를
2~4 문단으로 서술. 경력의 관통 테마를 한 문장으로 요약하며 마무리.

---

## 2막: 어떤 사람인가

### DNA
§3 DNA를 서술형으로 풀어쓴다.
각 DNA: 이름(볼드) → 왜 이렇게 명명했는지 1~2문장 → 구체적 사례.
DNA 전체를 관통하는 핵심 패턴을 문단 말미에 한 줄로 수렴.

### 결정의 순간들
§4 의사결정 사례를 에피소드처럼 서술.
각 사례: 상황→선택→결과를 3~5문장으로.
공통 의사결정 루프를 "이 사람은 ___→___→___를 반복한다"로 수렴.

### 강점과 약점
§5를 산문으로 풀어쓴다.
동전의 양면("추진력 ↔ 독단")은 같은 문단에서 대비.
"이 사람의 가장 큰 무기는 ___이다. 그러나 ___."

---

## 3막: 지금 어디로 가는가

### 현재 컨텍스트
§6 동향을 시간순 서술 (타임라인 테이블 1개 허용).
향후 예상 방향 2~3문장.

### 접근법
§7 High Fit·Low Fit을 산문으로 서술.
마지막에 추천 접근법을 1~2문장으로 수렴.

---

> 전체 확신도를 한 줄로 선언.
```

## §4 DEPTH CONTROL

| 깊이 | 트리거 | 축 수 | DNA 수 | 소스/축 | research-frame | 변환 산출물 |
|------|--------|-------|--------|---------|---------------|-----------|
| Quick | "간단히" "빠르게" | 3축(①③⑦) | 3대 | 2+ | §2 HOW만 경량 적용 | 프롤로그+1막+2막(DNA만) |
| Standard | 기본값 | 6축(①~⑤⑦) | 5대 | 3+ | §1→§2→§3 풀 적용 | 3막 전체+에필로그 |
| Deep | "깊이" "상세히" | 7축 전체 | 7대 | 5+ | §1→§2→§3 풀 + L2 교차분석 | 3막 전체+에필로그 |

**RULE**: 모든 깊이에서 `Skill("research-frame")` 호출 필수. Quick은 §2 HOW 경량 적용(가설·SCOPE 생략 가능). **모든 깊이에서 §3.5 리포트 변환 필수** — §3 중간 산출물을 최종 리포트로 직접 출력 금지.

## §5 QUALITY RULES

| 규칙 | 기준 |
|------|------|
| **증거 없는 주장 금지** | DNA·강점·약점 모두 구체적 사례 연결 필수 |
| **출처 인라인** | 서술 흐름 속에 "(매체명, 날짜)" 형식. 별도 소스 섹션 만들지 않음 |
| **시점 표기** | "현재" 정보는 조사 시점 명시 (예: "2025.04 기준") |
| **호불호 균형** | 긍정·부정 평가를 동등하게 수집. 한쪽 편향 시 명시 |
| **이미지** | 웹서치로 대표 이미지 URL 1~2개 확보. 미확보시 이미지 섹션 자체 생략 |
| **엘베피치 QC** | 리포트 완성 후 프롤로그가 2막 DNA·강약점과 정합하는지 재검증 |
| **서사 톤 검증** | 최종 리포트에서 테이블 3개 초과 또는 불릿 존재 시 산문으로 재작성 |

## §6 GOTCHAS

| 실수 | 방지책 |
|------|--------|
| 축 간 오염 (②에서 ④ 내용 혼입) | 축별 독립 조사 → 이후 교차 |
| DNA를 직함으로 대체 ("CEO DNA") | DNA는 행동 패턴이지 직함이 아님 |
| 강점만 나열, 약점 회피 | 강점·약점 동수 원칙 (±1 허용) |
| 오래된 정보로 "현재" 서술 | ⑦축은 반드시 최근 1년 이내 소스 |
| 엘베피치가 DNA와 불일치 | 변환 후 프롤로그↔2막 정합성 체크 |
| **§3 중간 산출물 직접 출력** | §3.5 변환 없이 §3을 최종 리포트로 제출 금지. 가장 흔한 실수 |
| **테이블 과잉** | 프로필 카드·타임라인 외 테이블 금지. 위반 시 산문 전환 |
